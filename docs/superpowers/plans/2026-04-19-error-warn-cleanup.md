# Error / Warn Cleanup Implementation Plan

> **RETIRED (2026-04-19).** This plan is a frozen-in-time execution record. The `ModLogger.Error(...)` public API was retired 2026-04-19; `validate_content.py` Phase 11 now enforces its absence in `src/`. For the current logging API, see AGENTS.md "Code Standards" + CLAUDE.md. For design rationale, see the (also frozen) spec at [`../specs/2026-04-19-error-warn-cleanup-design.md`](../specs/2026-04-19-error-warn-cleanup-design.md).

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Drop `ModLogger.Error(...)` from the public API by reclassifying all 363 call sites into `Surfaced` / `Caught` / `Expected`, retain `ModLogger.Warn(...)` as the legitimate log-only primitive, and add a permanent `validate_content.py` Phase 11 gate that prevents `Error` from being reintroduced.

**Architecture:** Three independently-revertible commits on branch `development`. Commit 1 is the mechanical sweep (~302 sites, transform `Error(cat, msg, ex)` → `Caught(cat, msg, ex)` and inline any `LogCategory` const arguments to literal strings). Commit 2 is the judgment pass over the 61 no-exception `Error(cat, msg)` sites + a capped opportunistic `Warn` audit. Commit 3 deletes the `Error` method from `ModLogger` and adds the Phase 11 regression gate. Each commit is gated by `dotnet build -c "Enlisted RETAIL" -p:Platform=x64` (`0 Warning(s) 0 Error(s)`) and `python Tools/Validation/validate_content.py` (no new errors beyond the 2 baseline `camp_decisions.json` tooltip errors). New C# files are not introduced by this plan, so the CLAUDE.md Rule 2 `<Compile Include=...>` step is unnecessary.

**Tech Stack:** .NET Framework 4.7.2, C#, Bannerlord v1.3.13 modding APIs, Python 3 for validation, PowerShell/bash for tooling.

**Design spec:** [docs/superpowers/specs/2026-04-19-error-warn-cleanup-design.md](../specs/2026-04-19-error-warn-cleanup-design.md)

## Notes for the implementer

- **Shell.** Runs in bash on Windows. Common GNU utilities (`cat`, `head`, `tail`, `grep`, `which`) are NOT on PATH — use the Read / Grep / Glob tools instead. `dotnet`, `git`, `python` are not on default PATH; prepend if needed:
  ```bash
  export PATH="/c/Program Files/dotnet:/c/Program Files/Git/cmd:$PATH"
  ```
  Python lives at `/c/Python313/python.exe` if `python` isn't found.

- **Build syntax.** The `Enlisted RETAIL` configuration name has a space, which trips bash's argument parser unless quoted. Use:
  ```bash
  dotnet build Enlisted.sln -c "Enlisted RETAIL" -p:Platform=x64
  ```
  Close `BannerlordLauncher` before every build — it holds `Enlisted.dll` open and the post-build copy fails with MSB3021.

- **Multi-line commit messages.** No `cat <<EOF` heredocs (no `cat` on PATH). Write the message to a temp file (e.g., `/c/Users/coola/commit_msg.txt`) and pass via `git commit -F <path>`. Delete the temp file after committing.

- **Staging.** Use explicit `git add <path>` per file, never `git add -A`. Another AI session may have in-flight edits in `Tools/normalize_crlf.ps1` / `Tools/check_crlf.ps1` / older spec drafts — those are not ours.

- **Concurrent edits.** Before each commit, run `git status` and confirm only your intended files are staged.

- **Decompile.** This plan does not require any new TaleWorlds API verification — `ModLogger.Caught` / `Surfaced` / `Expected` are the mod's own API, already in use throughout the codebase. The decompile (sibling of repo root, `../Decompile/`) is not relevant here.

- **CRLF.** No new C# / csproj / sln / ps1 files are created. CRLF normalization is unnecessary. Markdown / Python edits use the Edit tool, which preserves existing line endings.

- [ ] **Step 2: Per file, transform every with-exception Error call**

For each file in the batch, walk every `ModLogger.Error(...)` call that takes an exception (third positional argument named `ex`, `patchEx`, `inner`, etc.) and apply the Edit tool with this rule:

**Pattern with literal category:**
```csharp
// Before
ModLogger.Error("Bootstrap", "Failed to apply deferred patch", ex);

// After
ModLogger.Caught("Bootstrap", "Failed to apply deferred patch", ex);
```

**Pattern with `LogCategory` const:**
```csharp
// Before (LogCategory = "Naval")
ModLogger.Error(LogCategory, "Failed to assign ship", ex);

// After
ModLogger.Caught("Naval", "Failed to assign ship", ex);
```

**Pattern with `ex.InnerException`:**
```csharp
// Before
ModLogger.Error("Bootstrap", "Inner exception during patch", patchEx.InnerException);

// After
ModLogger.Caught("Bootstrap", "Inner exception during patch", patchEx.InnerException);
```

**Pattern with `$"..."` interpolation in the message:**
```csharp
// Before
ModLogger.Error("Bootstrap", $"Failed to patch {type.Name}", patchEx);

// After
ModLogger.Caught("Bootstrap", $"Failed to patch {type.Name}", patchEx);
```

**Important:** Do NOT touch sites that have only two arguments (no exception) — those are commit 2's scope.
**Important:** Do NOT delete the `private const string LogCategory = "..."` line, even if all migrated calls in that file inlined the literal. Other call sites (`ModLogger.Debug(LogCategory, ...)`, `ModLogger.Info(LogCategory, ...)`, `ModLogger.Warn(LogCategory, ...)`) may still reference it. A separate cleanup pass can prune orphan consts later.

_MODLOGGER_ERROR_RE = re.compile(r"\bModLogger\.Error\s*\(")


def _strip_csharp_comments(text: str) -> str:
    """Strip C# // line comments and /* ... */ block comments from source
    text, replacing them with whitespace so line numbers are preserved.
    Mirrors generate_error_codes.py:strip_comments — kept inline here to
    avoid a sys.path import dance."""
    block = re.compile(r"/\*.*?\*/", re.DOTALL)
    line = re.compile(r"//[^\n]*")
    def blank(m):
        return "".join("\n" if ch == "\n" else " " for ch in m.group(0))
    text = block.sub(blank, text)
    text = line.sub(blank, text)
    return text


def validate_no_modlogger_error_calls(ctx: ValidationContext):
    """Phase 11: confirm no ModLogger.Error(...) calls remain in src/.

    The Error API was retired on 2026-04-19. Every error-tier log must
    use ModLogger.Surfaced / Caught / Expected. See
    docs/superpowers/specs/2026-04-19-error-warn-cleanup-design.md."""
    print("[Phase 11] Validating no ModLogger.Error(...) calls remain...")
    src_root = Path(__file__).parent.parent.parent / "src"
    if not src_root.is_dir():
        ctx.add_issue(
            "error", "logging-api",
            f"src/ directory not found at {src_root}", str(src_root),
        )
        return

    found = 0
    for cs_file in src_root.rglob("*.cs"):
        try:
            text = cs_file.read_text(encoding="utf-8")
        except UnicodeDecodeError:
            text = cs_file.read_text(encoding="utf-8-sig")
        stripped = _strip_csharp_comments(text)
        for lineno, line in enumerate(stripped.splitlines(), 1):
            if _MODLOGGER_ERROR_RE.search(line):
                ctx.add_issue(
                    "error", "logging-api",
                    "ModLogger.Error is retired. Use Surfaced / Caught / Expected. "
                    "See docs/superpowers/specs/2026-04-19-error-warn-cleanup-design.md.",
                    str(cs_file.relative_to(src_root.parent)), str(lineno),
                )
                found += 1
    if found == 0:
        print("  OK: zero ModLogger.Error(...) call sites in src/.")

## Rollback playbook

The three commits are independently revertible by design.

- **Revert commit 3 (deletion + gate)** restores `ModLogger.Error(...)` and removes the Phase 11 gate. Commits 1 and 2 (the migrated call sites) remain functional — they don't depend on the API still existing.
- **Revert commit 2 (judgment)** restores the 61 no-exception `ModLogger.Error(...)` sites. The build will then fail (the API is deleted in commit 3) — so revert commit 3 first, then commit 2.
- **Revert commit 1 (mechanical)** restores the 302 with-exception sites. Same dependency: revert 3 then 2 then 1.

Practical rule: if a regression appears post-merge, revert in reverse order from the top.

```bash
git revert <sha-of-commit-3>
git revert <sha-of-commit-2>
git revert <sha-of-commit-1>
git push origin development
```

If the regression is isolated to a single migrated call site (e.g., one site really did need `Surfaced`, not `Caught`), prefer a small follow-up commit that re-classifies just that site over a wholesale revert.
